Project Motivation

Online shopping has become a vital part of our lives in the current digital era, providing accessibility and convenience unlike anything else. But consumers are growing more hesitant to share personal information due to worries about data security and privacy, particularly when it comes to sensitive information like credit card numbers. In light of the necessity for improved privacy compliance in e-commerce platforms, the goal of our project is to create a reliable and safe web application that prioritizes user privacy while managing payments and card information.

Introduction

As convenience meets consumer demand in an era of online commerce, protecting user data privacy and security has become a non-negotiable necessity. Tools Used

Flask: To construct the application's backend, we opted for Flask, a Python web framework that is both lightweight and adaptable. Flask gives us the tools we need to manage database interactions, routing, and request handling effectively.

Python: The main programming language used to create the backend logic for our online application is Python. It is the best option for web development tasks because of its ease of use, readability, and extensive library ecosystem.

MySQL: We chose the well-known open-source relational database management system MySQL for data management and storage. We can safely store user card information and carry out effective data retrieval and manipulation tasks thanks to MySQL.

HTML, CSS, and JavaScript: We used the traditional web trio for our application's front end: HTML for content organization, CSS for layout styling, and JavaScript for dynamic and interactive features.
